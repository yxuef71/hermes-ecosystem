breakdown

numtide / llm-agents.nix

This collection provides daily-updated Nix packages for a wide range of AI coding tools, including Hermes and Claude Code. It is ideal for users who need a unified proxy API and multi-agent orchestrators within a reproducible Nix-based environment.

rookiemann / portable-hermes-agent

Designed for Windows users, this no-install desktop agent provides a GUI and over 100 tools without requiring Docker or administrative rights. It integrates local model management, GPU monitoring, and a workflow engine for scheduled automation directly on the desktop.

TheAiSingularity / hermesclaw

This security-focused deployment uses NVIDIA OpenShell and kernel-level syscall blocking to provide hardware-enforced sandboxing for Hermes. It includes over 40 integrated tools and pre-configured security policies that range from strict inference to permissive system access.

xmbshwll / hermes-agent-docker

This project offers a streamlined Docker-based sandbox that includes both the official Hermes installation and mini-swe-agent. It supports persistent state through mounted directories and allows users to target specific Hermes versions using build arguments.

0xrsydn / nix-hermes-agent

This NixOS module enables declarative management of Hermes configurations, documents, and systemd services with a focus on reproducibility. It features secure secrets handling via sops-nix and supports a hybrid skill architecture for both managed and interactive installations.

JackTheGit / hermes-autonomous-server

Targeting production environments, this headless deployment configures Hermes Gateway as a reboot-safe systemd service on Ubuntu and Debian. It is optimized for autonomous execution, allowing the agent to perform recurring AI cron jobs and integrate with Nous Portal.

ellickjohnson / portainer-stack-hermes

This Docker Compose stack integrates with Portainer to deploy Hermes alongside a ttyd web terminal for browser-based interaction. It simplifies the deployment lifecycle by using GitHub Actions and GHCR for automated image builds.
