breakdown

mem0ai / mem0

A universal memory layer providing multi-level storage for users, sessions, and autonomous states. It differentiates itself with temporal reasoning for time-aware ranking and hybrid retrieval combining semantic, keyword, and entity signals.

volcengine / OpenViking

An open-source context database that treats agent memory as a filesystem using viking:// URIs. It utilizes a tiered L0/L1/L2 loading system to significantly reduce token consumption while improving task completion rates.

supermemoryai / supermemory

A high-performance memory engine featuring a custom vector graph and sub-300ms recall for massive token volumes. It offers native connectors for productivity tools like Notion and Google Drive to maintain auto-updating user profiles.

garrytan / gbrain

An opinionated brain for Hermes Agents that automatically extracts typed entity relationships into a self-wiring knowledge graph. It is designed for multi-tenant company deployments to synthesize cited answers across people and meeting notes.

vectorize-io / hindsight

A learning-centric memory system that implements specific retain, recall, and reflect workflows for long-term knowledge. It provides an easy-to-integrate LLM wrapper with enterprise storage support for PostgreSQL and Oracle AI Database.

campfirein / byterover-cli

A unique memory provider that manages context trees as git repositories, allowing for branching, merging, and versioning of markdown files. It features a high-speed 5-tier retrieval system with sub-100ms latency.

plastic-labs / honcho

A stateful agent library that focuses on extracting reasoning-based conclusions from conversations rather than simple vector matching. It tracks evolving entities across users, groups, and projects using hybrid BM25 and vector search.

greyhaven-ai / autocontext

A recursive self-improving harness that uses a five-role architecture to manage strategy and curation. It helps agents succeed on complex tasks by producing reusable playbooks and detailed execution traces.

stephenschoettler / hermes-lcm

A lossless context management plugin utilizing a hierarchical DAG structure for depth-aware compaction. It ensures no message is lost by combining a SQLite-backed raw history store with targeted retrieval tools.

AxDSan / mnemosyne

A zero-dependency memory system offering sub-millisecond latency through local SQLite storage. Its MEMORIA engine performs structured fact extraction and recursive gap analysis for high-speed retrieval.

elkimek / honcho-self-hosted

A self-hosted backend for the Honcho library that enables private, cross-session persistence for Hermes Agent. It supports local inference servers and OpenAI-compatible providers via a Docker-based setup.

yoloshii / ClawMem

A local-first memory engine that operates entirely on-device without cloud dependencies or API keys. It integrates via MCP servers and Claude Code hooks using a mix of BM25, vector search, and knowledge graphs.

plur-ai / plur

A shared memory layer for multi-agent systems that stores data as open-format YAML engrams. It employs ACT-R activation to strengthen relevant facts and decay stale information over time.

EfficientContext / ContextPilot

A context optimization tool that accelerates LLM inference by reordering and deduplicating shared context blocks. It boosts prefill throughput by 1.5–3× while maintaining accuracy across SGLang, vLLM, and Hermes Agent.

keepnotes-ai / keep

A reflective memory system featuring automated indexing for local directories, URLs, and multimedia. It uses ranked graph traversal and edge tags to create navigable relationships between pieces of information.

yepyhun / Brainstack

A memory kernel that prevents 'flat memory blob' issues by using a five-shelf architecture. It consolidates temporal continuity, graph truth, and corpus retrieval into a single plugin using SQLite and Kuzu.

amanning3390 / flowstate-qmd

An anticipatory memory system that prefetches context by analyzing agent session logs. It is an MCP-native server using SQLite-vec and local GGUF models for proactive RAG.

yantrikos / yantrikdb-hermes-plugin

A self-maintaining plugin that provides explainable retrieval by tagging every recall with a 'why_retrieved' reason. It natively tracks contradictions and manages agent-authored skills via an outcome ledger.

Ladybug-Memory / hermes-memory-plugin

A local-only plugin using a columnar .lbdb embedded graph database. It prioritizes the most critical information using a 1-10 importance ranking system during context prefetch.

alias8818 / hermes-tool-slimmer

A utility designed to reduce tool-schema token overhead through BM25 ranking and keyword selection. It optimizes how Hermes Agent handles native tools and MCP integrations to save context space.

MukundaKatta / hermes-agentmemory

A pull-model episodic memory plugin that ensures true deletion of records to prevent outdated summaries from affecting recall. It includes a trace.jsonl audit log for full transparency of memory operations.
