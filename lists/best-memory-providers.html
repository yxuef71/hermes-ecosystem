breakdown

mem0ai / mem0

As the official memory provider for Hermes Agent, Mem0 offers a universal memory layer with multi-level retention across users, sessions, and agent states. It utilizes temporal reasoning and hybrid retrieval to ensure agents maintain a time-aware understanding of both past and future events.

garrytan / gbrain

This opinionated memory system features a self-wiring knowledge graph that extracts entity relationships without requiring expensive LLM calls. It enhances context through a hybrid search mechanism that combines standard vector retrieval with backlink-boosted graph ranking.

vectorize-io / hindsight

Hindsight implements a 'learn-as-you-go' architecture focused on long-term retain, recall, and reflect workflows. It provides enterprise-grade storage options via PostgreSQL and Oracle, allowing agents to evolve their knowledge base through a simple two-line LLM wrapper.

plastic-labs / honcho

Honcho is a stateful memory library that uses asynchronous reasoning to extract conclusions from message traces and event logs. Its peer-centric model is designed to track evolving entities and multi-peer perspectives using a combination of BM25 and vector search.

greyhaven-ai / autocontext

This recursive harness uses five specialized agent roles to create an iterative self-improvement loop for complex tasks. It persists knowledge by generating structured playbooks and detailed execution traces specifically optimized for Hermes and Claude agents.

stephenschoettler / hermes-lcm

Hermes-LCM provides lossless context management by organizing conversation history into a hierarchical DAG structure. This allows for depth-aware compaction while ensuring raw history remains accessible via SQLite-backed tools for grepping and expanding context.

AxDSan / mnemosyne

Mnemosyne is a zero-dependency memory plugin for Hermes that prioritizes speed, offering sub-millisecond latency for all search and storage operations. It leverages SQLite with native vector support to provide a lightweight, local-first memory solution.

elkimek / honcho-self-hosted

This project provides a self-hosted backend for the Honcho library, enabling private cross-session memory for Hermes Agent users. It includes an automated setup script for Docker and Redis, supporting any OpenAI-compatible provider or local inference server.

yoloshii / ClawMem

ClawMem is a local-first context engine that operates entirely on-device without cloud dependencies or API keys. It integrates via MCP to provide hybrid retrieval across vector stores, BM25, and knowledge graphs for privacy-conscious agent deployments.

plur-ai / plur

Plur introduces a shared memory layer using an open YAML 'engram' format to facilitate zero-cost local learning. It employs ACT-R cognitive principles to strengthen relevant facts over time while allowing stale information to naturally decay.

EfficientContext / ContextPilot

Focused on inference efficiency, ContextPilot reduces prompt tokens by up to 36% through aggressive context deduplication. It significantly boosts prefill speeds for Hermes Agent and vLLM by optimizing prefix sharing and context optimization.

keepnotes-ai / keep

Keep acts as a reflective memory system that indexes code, documents, and media using OCR and automatic summarization. It builds navigable, bidirectional relationships between notes via edge tags, making it a robust knowledge base for agent tool integration.

amanning3390 / flowstate-qmd

Developed for the Hermes 2026 Hackathon, this anticipatory memory system prefetches context by monitoring agent session logs in real-time. It utilizes a local-first stack with SQLite-vec and GGUF models to provide low-latency RAG via an MCP-native server.
