breakdown

mem0ai / mem0

A universal memory layer providing multi-level retention for users, sessions, and autonomous agents. It utilizes hybrid retrieval and temporal reasoning to rank memories by date and state.

volcengine / OpenViking

A context database that treats memory and skills as a file system using viking:// URIs. It employs tiered L0/L1/L2 loading to significantly reduce token costs and improve task completion.

supermemoryai / supermemory

A high-performance memory engine featuring sub-300ms recall and byte-level deduplication for prompt-cache discounts. It includes native connectors for Google Drive, Notion, GitHub, and Gmail.

garrytan / gbrain

An opinionated brain for Hermes agents that uses a self-wiring knowledge graph to extract entity relationships without LLM calls. It includes a synthesis layer for cited answers and over 30 MCP tools.

vectorize-io / hindsight

A learning-centric memory system implementing retain, recall, and reflect workflows for long-term knowledge. It offers a simple LLM wrapper with enterprise storage support for PostgreSQL and Oracle AI Database.

campfirein / byterover-cli

A memory provider that treats context as a git repository, enabling branching, merging, and versioning of markdown trees. It features a fast 5-tier retrieval system with sub-100ms latency.

plastic-labs / honcho

A stateful agent library that extracts reasoning-based conclusions rather than relying on simple vector matching. It maintains evolving representations of users and projects in prompt-ready formats.

greyhaven-ai / autocontext

A recursive self-improvement harness that uses five specialized agent roles to generate structured playbooks. It ensures persistent knowledge transfer via CLI-first skill exports.

AxDSan / mnemosyne

A zero-dependency, SQLite-backed memory system designed for sub-millisecond performance. It combines vector similarity, FTS5, and importance scoring with native MCP support.

stephenschoettler / hermes-lcm

A lossless context management plugin using a hierarchical DAG to compact conversation history without losing detail. It allows agents to grep and expand historical nodes via a local SQLite database.

elkimek / honcho-self-hosted

A self-hosted backend for Honcho that enables private, cross-session persistence for Hermes Agent. It supports Docker, PostgreSQL, and Redis with compatibility for any OpenAI-style LLM.

yoloshii / ClawMem

A local-first context engine that eliminates cloud dependencies and API keys. It employs a hybrid retrieval approach combining BM25, vector search, and knowledge graphs.

Signet-AI / signetai

A portable identity and memory layer using SQLite and markdown for local data ownership. It provides cross-harness continuity across Hermes Agent, Claude Code, and OpenClaw.

plur-ai / plur

A shared memory layer using an open YAML engram format for multi-agent systems. It utilizes ACT-R activation to strengthen relevant facts and decay stale information over time.

EfficientContext / ContextPilot

A context optimization tool that increases cache hits and reduces prefill latency through prefix reordering and deduplication. It integrates natively with SGLang, vLLM, and the Hermes ecosystem.

yepyhun / Brainstack

A memory kernel that consolidates temporal continuity, graph truth, and corpus retrieval. It uses a multi-layered storage stack comprising SQLite, Kuzu, and Chroma.

keepnotes-ai / keep

A reflective memory system featuring ranked graph traversal and BM25 full-text search. It provides automated indexing for local directories, URLs, and multimedia files via edge tags.

yantrikos / yantrikdb-hermes-plugin

A self-maintaining plugin that provides explainable recall via why_retrieved tags. It automatically handles canonicalization, contradiction tracking, and recency ranking.

amanning3390 / flowstate-qmd

An anticipatory memory system that prefetches context by monitoring agent session logs. It is a local-first MCP server utilizing SQLite-vec and GGUF models.

Ladybug-Memory / hermes-memory-plugin

A local-only plugin using a columnar .lbdb embedded graph database. It employs importance-weighted recall and BM25 search to surface high-priority memories first.

alias8818 / hermes-tool-slimmer

A utility that reduces tool-schema overhead using BM25 ranking for keyword-based tool selection. It includes a dashboard to track prompt token savings for Hermes and MCP tools.

MukundaKatta / hermes-agentmemory

An episodic memory plugin featuring a pull-model for retrieval and true deletion of memories. It maintains a trace.jsonl audit log for every memory operation.
