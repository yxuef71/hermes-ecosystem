breakdown

mem0ai / mem0

A universal memory layer and official Hermes Agent provider offering multi-level retention for users, sessions, and autonomous agents. It utilizes hybrid retrieval and temporal reasoning to rank memories by date and state, available as a managed cloud or self-hosted OSS.

volcengine / OpenViking

An official Hermes Agent context database that uses a filesystem paradigm and tiered L0/L1/L2 loading to reduce token costs. It significantly improves task completion and token efficiency while providing visualized retrieval trajectories for debugging.

supermemoryai / supermemory

An official Hermes Agent memory engine featuring sub-300ms recall and byte-level deduplication for prompt-cache discounts. It automates fact extraction and user profile maintenance with native connectors for Google Drive, Notion, GitHub, and Gmail.

garrytan / gbrain

An opinionated brain for OpenClaw and Hermes Agents that synthesizes cited answers with explicit gap analysis of missing information. It features a self-wiring knowledge graph for complex entity traversal and supports MCP for integration with Cursor and Windsurf.

vectorize-io / hindsight

A long-term learning memory system implementing retain, recall, and reflect workflows. It provides a simple two-line LLM wrapper for automatic storage and retrieval, supporting enterprise backends like PostgreSQL and Oracle AI Database.

campfirein / byterover-cli

An official Hermes Agent provider that treats memory as a git repository, enabling branching, merging, and versioning over markdown context trees. It features a sub-100ms 5-tier retrieval system and built-in tools for file operations.

plastic-labs / honcho

A stateful agent memory library that extracts reasoning-based conclusions rather than relying on simple vector matching. It maintains evolving representations for users and projects, providing prompt-ready context for OpenAI and Anthropic formats.

greyhaven-ai / autocontext

A recursive self-improving context harness that uses five specialized agent roles to generate structured playbooks and traces. It enables persistent knowledge transfer through native CLI-first skill exports for Hermes Agent.

AxDSan / mnemosyne

A zero-dependency, sub-millisecond memory system for Hermes Agents utilizing a BEAM architecture with working, episodic, and triple-store memory. It is built on local-first SQLite with sqlite-vec and FTS5 for broad compatibility.

stephenschoettler / hermes-lcm

A lossless context management plugin that uses a hierarchical DAG structure for depth-aware context compaction. It preserves raw history in a SQLite store, allowing agents to search and expand compacted conversation nodes.

elkimek / honcho-self-hosted

A self-hosted backend for the Honcho library that enables private cross-session persistence for Hermes Agent. It supports local inference servers or any OpenAI-compatible provider and includes a Docker-based automated setup.

yoloshii / ClawMem

A local-first, on-device context engine that eliminates cloud dependencies and API keys. It employs hybrid retrieval combining BM25, vector search, and knowledge graphs, integrating via MCP servers and Claude Code hooks.

Signet-AI / signetai

A portable local-first identity and secrets layer that maintains state across Hermes Agent, Claude Code, and OpenClaw. It uses SQLite and markdown for data ownership, combining vector search, FTS5, and graph traversal for retrieval.

plur-ai / plur

A shared memory layer for multi-agent systems using an open YAML engram format to eliminate API costs. It utilizes ACT-R activation to strengthen relevant facts and decay stale ones via MCP tools for hybrid context injection.

EfficientContext / ContextPilot

A context optimization tool that increases cache hits and reduces prefill latency by reordering and deduplicating prefixes. It provides native plugin support for Hermes Agent and OpenClaw to accelerate long-context LLM inference.

yepyhun / Brainstack

A memory kernel stack that consolidates temporal continuity, graph truth, and corpus retrieval. It utilizes a multi-layered storage approach with SQLite, Kuzu, and Chroma to provide bounded evidence packing that reduces prompt noise.

keepnotes-ai / keep

A reflective memory system featuring semantic search with BM25 and ranked graph traversal. It provides automated indexing for local directories, URLs, and multimedia files, using edge tags to create navigable relationships.

yantrikos / yantrikdb-hermes-plugin

An embedded, self-maintaining memory plugin that provides explainable recall via why_retrieved tags. It includes primitives for contradiction tracking and recency ranking without requiring a separate server or GPU.

amanning3390 / flowstate-qmd

An anticipatory memory system that prefetches context by monitoring agent session logs. It uses local-first storage with SQLite-vec and GGUF models, operating as an MCP-native server for coding agent clients.

alias8818 / hermes-tool-slimmer

A utility that reduces tool-schema overhead by ranking tools using BM25 and keyword selection. It includes a dashboard to track token savings and supports custom tool profiles for inclusion or exclusion.

Ladybug-Memory / hermes-memory-plugin

A local-only plugin using an embedded .lbdb graph database with importance-weighted recall to surface high-priority memories. It supports BM25 keyword search and optional GLiNER2 entity extraction.

MukundaKatta / hermes-agentmemory

A pull-model episodic memory plugin that ensures complete deletion of memories without leaving persistent summaries. It provides a trace.jsonl audit log of every operation and on-demand summarization via the Claude model family.
