breakdown

mem0ai / mem0

As the official memory provider for Hermes Agent, Mem0 offers a universal memory layer that tracks user, session, and agent states across a managed cloud or self-hosted environment. It utilizes a hybrid retrieval system and single-pass fact extraction to minimize latency while ensuring no critical data is lost during the recall process.

garrytan / gbrain

This opinionated 'agent brain' features a self-wiring knowledge graph that automatically extracts entities and relationships from interactions. It differentiates itself by combining graph-based ranking with vector search and exposing over 30 MCP tools for deep integration with Hermes and coding assistants.

vectorize-io / hindsight

Hindsight implements a specialized 'retain, recall, and reflect' workflow that allows agents to learn and improve their responses over time. It provides a simple two-line LLM wrapper for automatic memory management and supports strict per-user isolation through custom metadata filtering.

plastic-labs / honcho

Honcho is a versatile library designed to maintain evolving state for users and groups through natural language querying of interaction histories. It serves as the foundational upstream technology for several Hermes memory wrappers, offering SDKs that format context for OpenAI-compatible environments.

greyhaven-ai / autocontext

This recursive context harness focuses on complex task success by using an iterative self-improvement loop to evaluate agent strategies. It generates structured playbooks and persistent knowledge traces, providing native integrations specifically for Hermes Agent and Claude Code.

stephenschoettler / hermes-lcm

Hermes-LCM is a lossless context management plugin that uses a Directed Acyclic Graph (DAG) to compact context without ever deleting raw message history. It stores data in SQLite and provides agents with specialized tools to grep, describe, and expand compacted conversation segments.

elkimek / honcho-self-hosted

This project provides a self-hosted backend for the Honcho memory system, enabling private cross-session persistence for Hermes Agent users. It is designed for easy deployment via Docker and supports any OpenAI-compatible provider or local inference server.

AxDSan / mnemosyne

Mnemosyne is a zero-dependency, SQLite-backed system designed for sub-millisecond memory retrieval in Hermes Agents. It features a unique three-tier BEAM architecture that manages working, episodic, and scratchpad memory through automated compression and 'sleep' cycles.

yoloshii / ClawMem

ClawMem is a local-first context engine that operates entirely on-device without cloud dependencies or API keys. It employs a sophisticated hybrid retrieval method involving causal graph traversal and BM25 to share memory across Hermes, OpenClaw, and Claude Code runtimes.

EfficientContext / ContextPilot

Focused on inference efficiency, ContextPilot optimizes prefix reuse through reordering and deduplication to reduce prompt tokens by up to 36%. It is an ideal choice for Hermes users looking to accelerate long-context workloads in vLLM, SGLang, or llama.cpp.

plur-ai / plur

Plur introduces a shared memory layer using an open YAML format called 'engrams' to facilitate memory exchange in multi-agent systems. It offers zero-cost local storage and automates memory injection for Hermes and Cursor using a blend of BM25 and embedding search.

keepnotes-ai / keep

Keep provides reflective memory by indexing code, documents, and media with automatic OCR and summarization capabilities. It uses edge tags to build bidirectional relationships between notes, making it a powerful knowledge-base companion for agents via MCP and CLI interfaces.

amanning3390 / flowstate-qmd

Developed for the Hermes 2026 Hackathon, this anticipatory memory system prefetches context by monitoring agent session logs in real-time. It relies on a local-first architecture using SQLite-vec and GGUF models to provide low-latency RAG capabilities via an MCP-native server.
